Thread Design & Concurrency and Light Weight Processes

Kernel Threads vs User Threads

‚öôÔ∏è Kernel Threads

‚úì Managed by Kernel
  • Created and scheduled directly by kernel
  • No library overhead - pure kernel operation
  • Run in kernel mode with full CPU privileges
‚úì Common Examples
  • kworker - background task handling
  • ksoftirqd - interrupt processing
  • kswapd - memory swapping daemon
  • pdflush - page cache flushing
‚úì Use Cases
  • Critical system operations
  • Interrupt handling
  • Memory management
  • I/O scheduling

üìö User Threads

‚úì Managed by Libraries
  • Created via libraries like pthreads (POSIX)
  • Mapped to kernel threads for execution
  • Run in user mode with restricted access
‚úì Common Creation
  • pthread_create() - POSIX threads
  • Web browsers & servers
  • Desktop GUI applications
  • Database connection handlers
‚úì Advantages
  • Lightweight - less memory overhead
  • Fast creation and switching
  • Shared memory synchronization
  • Responsive user experience
