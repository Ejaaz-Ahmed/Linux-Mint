Process Scheduling Architecture

Three-Level Scheduling System

📥 Long-Term Scheduler
Role: Admits new processes into the system Frequency: Seconds / Minutes Impact: Prevents system overload and maintains stability Key: Resource gating
🔄 Mid-Term Scheduler
Role: Manages memory swapping (RAM ↔ Disk) Frequency: Seconds Impact: Maintains system performance during high load Key: Memory balancing
⚡ Short-Term Scheduler (CFS)
Role: Selects processes for CPU execution Frequency: Milliseconds Impact: Responsive desktop experience Key: Fair CPU time allocation

Thread Design & Concurrency and Light Weight Processes

Kernel Threads vs User Threads

⚙️ Kernel Threads

✓ Managed by Kernel
  • Created and scheduled directly by kernel
  • No library overhead - pure kernel operation
  • Run in kernel mode with full CPU privileges
✓ Common Examples
  • kworker - background task handling
  • ksoftirqd - interrupt processing
  • kswapd - memory swapping daemon
  • pdflush - page cache flushing
✓ Use Cases
  • Critical system operations
  • Interrupt handling
  • Memory management
  • I/O scheduling

📚 User Threads

✓ Managed by Libraries
  • Created via libraries like pthreads (POSIX)
  • Mapped to kernel threads for execution
  • Run in user mode with restricted access
✓ Common Creation
  • pthread_create() - POSIX threads
  • Web browsers & servers
  • Desktop GUI applications
  • Database connection handlers
✓ Advantages
  • Lightweight - less memory overhead
  • Fast creation and switching
  • Shared memory synchronization
  • Responsive user experience
